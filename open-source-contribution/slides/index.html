          • What is open source?

          denoting software for which the original source code is made freely available and may be redistributed and modified.

          This means that anyone...

          • Can take the source code
          • Can modify it
          • Can distribute their own versions

            • Other benefits it provides

            • Cost benefits (flexibility and freedom but it requires maintenance and support)
            • Community support (peer review ensure security and accountability)
            • Quality of code
            • Transparency & reliability

                How do you become part of this community and contribute to cool projects?

                I will specifically be showing how you can contribute using Github.

                What is Github?

                • source version control
                • contains lots of repos
                • place where you can pull down a repo and contribute back

                  • picture of github

                  Your first contribution

                  • find something easy (docs, bug, etc)
                  • look for specific labels: first-timers-only, good first issue, low-hanging-fruit, beginner, easy
                  • find something you're passionate about
                  • things to consider: language, project, size

                    • Now you've found a bug, how do you contribute the fix back?

                    01 Fork the repo

                    02 Clone the repo locally

                    03 Get the project running and check out branch

                    04 Fix the issue

                    05 Commit and push fix

                    06 Create pull request comparing branch and master

                    07 Tests will run and pass or fail

                    08 One or more reviews are required

                    09 Changes made if necessary

                    10 Pull request gets merged to master

                    Tada! You've made your first pull request!

                    Common pitfalls:

                    • Not everything you find is open source (look for license)
                    • Establishing coding standards
